# sample-data — 케이스 배치 규칙

이 폴더에는 차지백 케이스가 **케이스 1건 = 폴더 1개** 로 들어갑니다.
이 워크숍은 **발급사(Issuer) 입장의 OB(아웃바운드) 1CB(First Chargeback) 이의신청**을 다룹니다.
즉, 하나카드가 자사 카드홀더를 대리해 환불 미이행·청구금액 상이·이중청구 등으로 **1CB를 개시**하는 업무이며,
실습 케이스도 모두 OB(발급사·1CB) 기준입니다.

## 폴더 구조

```
sample-data/
├── README.md                ← (이 파일)
├── case-anon-ob-01/         ← 커밋된 익명 더미 OB 케이스 (예시·Lab 3 동작 확인용)
│   ├── customer-email.txt        카드홀더 이의제기 메일 (비정형 한국어)
│   ├── txn-summary.txt           거래/취소 요약 (비정형)
│   └── output/                   AI 산출물 저장 위치 (.gitkeep 만 커밋)
│       └── (예: 1cb-draft.md  ← 참가자가 만든 스킬이 생성하는 1CB 영문 레터 초안)
│
└── (gitignored — SA가 배치하는 실 OB 케이스)
    └── sample/OB/
        ├── ..._0_고객제출자료_raw data_*       단계0 = 고객 raw 증빙
        └── ..._1_1CB등록자료_*_1CB Doc.pdf     단계1 = 실제 등록된 1CB Doc (정답지)
```

### 익명 더미 케이스 (`case-anon-ob-01`) — 유일한 커밋 대상

- **13.6 Credit Not Processed(취소/환불 미이행)** 를 모델로 한 **완전 가상** 케이스입니다.
- 시나리오: 카드홀더(홍길동)가 가상 호텔(`ABC Hotel`) 예약을 취소했고 호텔이 환불을 확인(VOID 표기·채팅 확인)했으나
  카드 환불이 미처리됨 → 1CB 청구 요청.
- 구성: `customer-email.txt`(비정형 이의제기 메일) + `txn-summary.txt`(거래/취소 요약·타임라인) + `output/`(빈 산출물 폴더).
- 용도: 참가자가 Lab 2에서 직접 만든 OB 차지백 스킬을 **Lab 3에서 동작 확인**하는 입력입니다.
- 케이스를 추가하려면 폴더를 새로 만들고 같은 식으로 파일을 넣습니다.
  한 폴더 안에는 그 케이스의 입력 자료와 `output/`(산출물)만 둡니다.

## 🔒 실 OB 케이스(`sample/OB/`)는 gitignored — 절대 커밋 금지

- 실 OB 케이스는 **실(비식별화) 고객 데이터**라 **gitignore** 되어 있고, **SA가 워크숍 환경에 직접 배치**합니다.
  저장소에는 들어가지 않습니다.
- **`case-anon-ob-01`만 커밋 대상**(익명 더미)입니다. 실 데이터·VISA 규정 원문은 절대 커밋하지 않습니다.
- **단계 구성:** `..._0_고객제출자료_raw data_*`(단계0 = 고객 raw 증빙) + `..._1_1CB등록자료_*_1CB Doc.pdf`(단계1 = **실제 등록된 1CB Doc = 정답지**, 톤·구조 few-shot 참고용).
- **이미지 증빙 포함:** 가맹점/호텔 화면 캡처(VOID 표기)·취소 영수증 사진·채팅 스크린샷 등이 섞여 있어 **비전(이미지 읽기)** 으로 핵심정보를 추출합니다. OB 전용 VISA 규정집도 별도(`visa-core-rules ... 18-october-2025.pdf`).

## 🔴 익명·마스킹 규칙 (반드시 지킴)

이 POC는 **익명 더미 데이터만** 사용합니다. 실제 고객 개인정보(PII)는 절대 넣지 않습니다.

| 항목 | 규칙 | 예시 |
|------|------|------|
| 이름 | 가짜 이름 사용 | 홍길동, Gildong Hong |
| 카드번호 | `1111`로 마스킹 | `1111-1111-1111-1111` |
| 주민등록번호 | **넣지 않음** | (제거) |
| 전화/이메일/주소 | 가짜 더미값 | `dummy@example.com` |
| 가맹점/호텔명 | 가짜 또는 일반화 | "ABC Hotel", "OO호텔" |

- **주민번호 + 카드번호 조합 식별 금지**: 두 값이 함께 있으면 익명이라도 넣지 않습니다.
- 입력 자료에 실제 PII가 보이면 산출물/인덱싱 전 **마스킹 필수**입니다. 미상 식별값은 날조 없이 `[CONFIRM]`/`[MASKED]` 로 표기합니다 (CLAUDE.md 참고).
- 실제(비식별화) 케이스로 검증할 때도 별도 보안 검토를 거친 데이터만 사용합니다.
