# sample-data — 케이스 배치 규칙

이 폴더에는 차지백 케이스가 **케이스 1건 = 폴더 1개** 로 들어갑니다.
이 워크숍의 기본은 **인바운드(하나카드=매입사) 재반박**이며, 실습 케이스도 모두 인바운드입니다.

## 폴더 구조

```
sample-data/
├── README.md            ← (이 파일)
├── case-anon-01/        ← 커밋된 익명 더미 케이스 (예시·동작 확인용)
│   ├── customer-email.txt     이의제기 메일 (비정형)
│   ├── txn-summary.txt        거래/서류 요약 (비정형)
│   └── output/                AI 산출물 저장 위치
│       └── representment-draft.md  draft-writer가 만든 재반박 초안 (1CB면 1cb-draft.md)
│
└── (gitignored — 강사가 배치하는 실 인바운드 케이스 5종)
    ├── case-12.5/   ├── case-12.6/   ├── case-13.1/   ⭐가이드 케이스
    ├── case-13.3/   └── case-13.7/
        각 폴더 = VISA접수내역 + 분쟁 doc + 가맹점자료 (+ output/)
```

- **각 실 케이스 폴더 구성:** ① VISA 접수내역(intake — Issuer/Acquirer·사유코드 포함), ② 분쟁 doc, ③ 가맹점 자료(재반박 증빙). 참고: `_doc.pdf`가 가맹점 재반박 증거인 경우가 있어 파일명이 분쟁서류처럼 보여도 내용 확인 필요.
- **⭐ 13.1 가이드 케이스:** `case-13.1` 은 워크숍 시연용 표준 케이스입니다 (사유코드 13.1, 인바운드/매입사 재반박). 흐름·산출물 설명은 이 케이스 기준.
- 케이스를 추가하려면 폴더를 새로 만들고 같은 식으로 파일을 넣습니다. 한 폴더 안에는 그 케이스의 입력 서류와 `output/`(산출물)만 둡니다.

## 🔒 실 케이스 폴더는 gitignored — 절대 커밋 금지

- `case-12.5 / case-12.6 / case-13.1 / case-13.3 / case-13.7` 5종은 **실(비식별화) 고객 데이터**라 **gitignore** 되어 있고, **강사가 워크숍 환경에 직접 배치**합니다. 저장소에는 들어가지 않습니다.
- **`case-anon-01`만 커밋 대상**(익명 더미)입니다. 실 데이터·VISA 규정 원문은 절대 커밋하지 않습니다.

## OB (아웃바운드 / 1CB 예시 — 보조)

- **위치:** `sample/OB/` (gitignored — 강사가 배치, 절대 커밋 금지).
- **용도:** **보조(아웃바운드/발급사) 1CB 모드**의 예시·검증용입니다. 하나카드가 발급사인 건으로, 카드홀더 대리로 매입사에 1CB를 청구한 실제 케이스(가명 처리: 12.5 청구금액상이 / 12.6 이중청구 / 13.6 취소불이행)입니다.
- **단계 구성:** `..._0_고객제출자료_raw data_*`(단계0 = 고객 raw) + `..._1_1CB등록자료_*_1CB Doc.pdf`(단계1 = **실제 등록된 1CB Doc = 정답지**, 톤·구조 few-shot 참고용).
- **이미지 증빙 포함:** 경찰리포트 스캔(JPG)·호텔 스크린샷(PNG/JPG)이 섞여 있어 **비전(이미지 읽기)** 으로 핵심정보를 추출합니다 (IB는 전부 텍스트 PDF였음). OB 전용 VISA 규정도 별도(`visa-core-rules ... 18-october-2025.pdf`).
- **⚠️ 이 워크숍의 주력은 여전히 IB(인바운드/매입사 재반박)** 이며, OB는 1CB 모드의 예시·검증 보조일 뿐입니다.

## 🔴 익명·마스킹 규칙 (반드시 지킴)

이 POC는 **익명 더미 데이터만** 사용합니다. 실제 고객 개인정보(PII)는 절대 넣지 않습니다.

| 항목 | 규칙 | 예시 |
|------|------|------|
| 이름 | 가짜 이름 사용 | 홍길동, Gildong Hong |
| 카드번호 | `1111`로 마스킹 | `1111-1111-1111-1111` |
| 주민등록번호 | **넣지 않음** | (제거) |
| 전화/이메일/주소 | 가짜 더미값 | `dummy@example.com` |
| 가맹점명 | 가짜 또는 일반화 | "ABC Online Store" |

- **주민번호 + 카드번호 조합 식별 금지**: 두 값이 함께 있으면 익명이라도 넣지 않습니다.
- 입력 서류에 실제 PII가 보이면 document-analyzer가 경고하도록 되어 있습니다 (CLAUDE.md 참고).
- 실제(비식별화) 케이스로 검증할 때도 별도 보안 검토를 거친 데이터만 사용합니다.
